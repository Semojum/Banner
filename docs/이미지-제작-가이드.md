# 세모점 홈페이지 — placeholder 이미지 제작 가이드

`index.html`에 아직 실제 이미지가 안 들어간 **5자리**를 채우기 위한 촬영/제작 체크리스트.
앱이 대부분 완성돼 있으므로 **실제 앱 스크린샷** 기준으로 정리한다.

> 준비가 끝나면 파일을 `IMG/` 폴더에 넣고 알려주면, placeholder `<div>`를 `<img>`로 교체(+ `data-ix` 라벨 정리)한다.

---

## 0. 공통 규칙

- **하나의 데모 문서로 통일** — 5장이 같은 소재를 보여주면 스토리가 이어지고 신뢰도가 오른다.
  권장 소재: **「고등학교 수학 I — 이차방정식」 교재 1페이지(2단 레이아웃)**
  - 본문(좌·우 2단) + **수식** 근의 공식 `x = (-b ± √(b²-4ac)) / 2a`
  - **표**: 판별식 D 부호별 근의 개수(`D>0` / `D=0` / `D<0`)
  - **그림**: 포물선 그래프 `y = x²-2x-3`
- **더미 텍스트는 실제 한국어로** (로렘입숨 금지 — 교재/점자 맥락이라 가짜 티 나면 신뢰 하락)
- **색·폰트**: 사이트 팔레트 유지 — 흰 배경 `#FFFFFF`/`#F7F9FD`, 블루 `#4F7DF0`·`#2D4FCC`, 코랄 `#FF8B6B`
- **해상도**: 표시 크기의 **2배(레티나)** 로 캡처 → PNG, 가능하면 200KB 이내로 최적화
- 모서리/그림자는 페이지 CSS가 입히므로 **스크린샷 자체는 여백 없이 화면만** 깔끔하게 크롭

---

## 1. 자리별 체크리스트

| 파일명 | 위치(섹션) | 표시 크기 | 권장 캡처(2x) |
|---|---|---|---|
| `IMG_05.png` | Solution 히어로 | 1140×360+ | **2280×900** (와이드) |
| `IMG_06.png` | 핵심 1 — 레이아웃 인식 | ~540×360 | **1080×720** |
| `IMG_13.png` | 모드① 원문 정리 | ~360×200 | **720×400** |
| `IMG_14.png` | 모드② 점자 파일 생성 | ~360×200 | **720×400** |
| `IMG_15.png` | 모드③ 원클릭 점역 | ~360×200 | **720×400** |

> ⚠️ `IMG_09.png`·`IMG_10.png`는 이미 다른 화면(에디터·OCR)에 쓰이는 중. 새 파일은 위 번호(`13/14/15`)로.

---

### IMG_05 — Solution 히어로 (서비스 메인 화면)
- **찍을 화면**: 앱 메인 대시보드 또는 업로드 화면 (첫인상 컷, 가장 공들일 것)
- **띄울 데이터**: 프로젝트 목록 3~4개 + 상태 배지
  - `고1 수학 1단원.pdf — ✅ 점역 완료`
  - `사회탐구 교재.hwp — 🟡 검토 중 62%`
  - `과학 실험서.pdf — ⬆ 업로드됨`
  - 우측에 강조된 **`문서 업로드`** 버튼 / 드래그-드롭 존이 보이게
- **구도**: 앱 창 전체, 정면(플랫) 와이드 가로
- **체크**: 개인정보/실명/실데이터 없는지, 빈 화면(empty state) 아닌지

### IMG_06 — 핵심 1: 복합 레이아웃 인식
- **찍을 화면**: 문서 위에 **인식 영역/읽기 순서가 표시되는 화면**
  - 앱에 해당 오버레이 UI가 있으면 그대로 스크린샷
  - 없으면 이 컷만 간단한 다이어그램으로 대체(색 박스 + 번호 배지)
- **띄울 데이터**: 2단 데모 문서에 색 박스로 `본문`·`표`·`그림`·`수식` 라벨 + **읽기 순서 번호 ①②③…**
  좌단 ①② → 표 ③ → 우단 ④⑤ → 그림 ⑥ 순으로 올바르게 흐르게
- **구도**: 단일 페이지 어노테이션

### IMG_13 — 모드① 원문 정리 (PDF·HWP·IMG → TXT)
- **찍을 화면**: 원문 → 정리된 텍스트 결과 (좌 원문 / 우 정리 텍스트)
- **띄울 데이터** (우측 결과):
  - 수식: `근의 공식: x = (-b ± √(b²-4ac)) / 2a`
  - 표가 텍스트로 풀림 + `[표 설명 초안: 판별식 D의 부호에 따라 …]`
  - 그림: `[그림 설명: 아래로 볼록한 포물선, x축과 두 점에서 만남]`

### IMG_14 — 모드② 점자 파일 생성 (TXT → BRF)
- **찍을 화면**: 검토 완료 텍스트 → 점자(BRF) 출력 (좌 텍스트 / 우 점자 미리보기)
- **띄울 데이터**: 한 줄 한글 + 대응 점자 점형, 수식의 수학 점자 표기, 파일명 `고1수학_1단원.brf`
  - "점자 프린터·점자정보단말기 호환" 표시 있으면 함께

### IMG_15 — 모드③ 원클릭 점역 (PDF·HWP·IMG → BRF)
- **찍을 화면**: 업로드 한 번 → 점자 파일까지 자동 (단일 플로우)
- **띄울 데이터**: `수학교재.pdf` → ⚡ → `수학교재.brf 준비됨`, 진행바 100%/「점역 완료」, 다운로드 버튼
  - (선택) "기존 대비 N배 빠름" 같은 절감 지표

---

## 2. 준비 완료 후

1. 파일을 `IMG/`에 저장 (위 파일명 그대로)
2. 알려주면 `index.html`의 placeholder 5개를 실제 `<img>`로 교체 + `Dockerfile` COPY 확인 + 재배포

> 이 문서는 `.gcloudignore`/`.dockerignore`의 `*.md` 규칙으로 **배포 이미지엔 포함되지 않는다**(저장소 문서용).
