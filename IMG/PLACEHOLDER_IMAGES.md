## 2. 채워야 할 플레이스홀더 5개

### ① IMG-05 — 솔루션 페이지 히어로 *(가장 중요)*
- **위치:** 솔루션 상단 "수식·표·그림도, 더 쉽고 빠르게" 아래 (`index.html:524`)
- **데이터:** 세모점 서비스 **메인 화면 스크린샷** (대시보드 또는 파일 업로드 화면)
- **구도:** 가로 와이드, 폭 1140px / 높이 360px 이상 → **약 16:9 ~ 16:7 풀폭 배너**. 화면 전체가 한눈에 보이는 정면 캡처. 실제 UI가 없으면 브라우저 목업(맥 윈도우 프레임) 안에 넣는 구도 권장.

### ② IMG-06 — 핵심1 "복합 레이아웃을 사람처럼 읽습니다"
- **위치:** 핵심1 텍스트 우측 (`index.html:548`)
- **데이터:** **복합 레이아웃 인식 예시** — 여러 단(컬럼)·표·읽기 순서를 색/번호로 구분해 인식한 화면 또는 다이어그램
- **구도:** 절반 폭(약 500~540px) / 높이 300px 이상 → **약 3:2 가로**. 원문 위에 영역 박스(단=파랑, 표=주황, 읽기순서=번호 화살표)를 오버레이한 주석형 구도가 효과적.

### ③ IMG-09 — 처리 모드 ‘원문 정리’
- **위치:** "처리 모드" 첫 번째 카드 상단 (`index.html:590`)
- **데이터:** '원문 정리' 모드 화면 (PDF·HWP·IMG → TXT)

### ④ IMG-10 — 처리 모드 ‘점자 파일 생성’
- **위치:** "처리 모드" 두 번째 카드 상단 (`index.html:594`)
- **데이터:** '점자 파일 생성' 모드 화면 (TXT → BRF)

### ⑤ IMG-11 — 처리 모드 ‘원클릭 문서 점역’
- **위치:** "처리 모드" 세 번째 카드 상단 (`index.html:598`)
- **데이터:** '원클릭 문서 점역' 모드 화면 (PDF·HWP·IMG → BRF)

> **③④⑤ 공통 구도:** 카드 상단 가로 띠, 각 폭 약 360px / 높이 150px 이상 → **약 16:9 가로 크롭**. 3장은 **톤·앵글·여백을 통일**해야 카드가 정렬돼 보임. 각 모드 핵심 영역(업로드 / 변환 / 결과)이 잘리지 않게 중앙 배치.

## 3. 권장 사양 (공통)

- **포맷:** PNG(스크린샷·UI) / JPG(사진). 실제 UI 스크린샷이 가장 신뢰도 높음.
- **해상도:** 표시 크기의 **2배(레티나)** — 예: IMG-05는 약 2280px 폭 캡처.
- **교체 방법:** 점선 `div.imgph`를 아래 형태로 교체. CSS(`.imgph.real`)는 이미 준비됨.

```html
<div class="imgph real">
  <img src="IMG/파일명" alt="설명" width="너비" height="높이" loading="lazy" decoding="async">
</div>
```
